# 📝 프로젝트명: 국어 단어 마스터 1000

## 1. 프로젝트 개요
* **목표:** 국어 단어 1,000자 완전 학습 및 진도 관리 웹 서비스 구축
* **핵심 가치:** AI(인공지능)와 연동하여 단순 암기를 넘어선 '문맥과 뉘앙스' 맞춤형 학습
* **배포 환경:** Cloudflare Pages (프론트엔드) + Cloudflare Workers (AI 연결 서버)
* **개발 방식:** Antigravity (복잡한 빌드 과정 없이 HTML/CSS/Vanilla JS로 가볍게 구축)

---

## 2. 파일 및 폴더 구조 (예상)
작업 폴더 안에 아래와 같은 구조로 파일을 생성하고 개발을 진행합니다.

```text
📁 k-word-master/
 ├── 📄 index.html      (메인 화면 및 UI 구조)
 ├── 📄 style.css       (화면 디자인)
 ├── 📄 app.js          (프론트엔드 핵심 로직 및 화면 제어)
 ├── 📄 data.js         (초기 1000자 단어 데이터 보관용)
 └── 📄 worker.js       (Cloudflare Workers에 올릴 AI 통신용 백엔드 코드)
```

---

## 3. 핵심 기능 정의

### A. 대시보드 (진도 관리)
* **프로그레스 바 (Progress Bar):** 1000자 중 현재 마스터한 단어 비율을 시각적으로 표시.
* **상태별 단어 목록:** 1. `대기 (Pending)`: 아직 한 번도 보지 않은 단어
  2. `학습 중 (Learning)`: 학습을 시작했지만 완벽히 외우지 못한 단어
  3. `마스터 (Mastered)`: 완전히 내 것으로 만든 단어

### B. 스마트 단어장 (Study Mode)
* **단어 카드 UI:** 앞면에는 단어, 뒷면에는 뜻이 나오는 플래시 카드 형식.
* **AI 예문 생성 기능:** 버튼을 누르면 AI가 해당 단어를 활용하여 실생활에서 쓰이는 자연스러운 예문을 즉석에서 생성하여 보여줌.

### C. 퀴즈 및 복습 (Quiz Mode)
* **자동 퀴즈 출제:** '학습 중' 상태인 단어들을 모아 4지 선다형 퀴즈 게임 형식으로 제공.
* **오답 관리:** 틀린 단어는 AI가 뉘앙스 차이나 틀린 이유를 설명해주고, 다시 '학습 중' 상태로 유지.

## 4. 데이터베이스 구조 (Local Storage 활용)
프론트엔드(`app.js`)에서 관리할 단어 데이터의 JSON 배열 구조입니다. 이 데이터는 브라우저의 `localStorage`에 저장되어 새로고침해도 날아가지 않습니다.

```json
[
  {
    "id": 1,
    "word": "시나브로",
    "meaning": "모르는 사이에 조금씩 조금씩.",
    "status": "learning", 
    "reviewCount": 2
  },
  {
    "id": 2,
    "word": "안다미로",
    "meaning": "담은 것이 그릇에 넘치도록 많이.",
    "status": "pending",
    "reviewCount": 0
  }
]
```

## 5. 단계별 개발 계획 (마일스톤)

### Phase 1: 기본 UI 뼈대 만들기 (HTML/CSS)
* [ ] `index.html`에 헤더, 진행률 바, 단어 카드가 들어갈 영역 퍼블리싱.
* [ ] Tailwind CSS V4로 깔끔하고 보기 편한 디자인 적용. 폰트는 Noto Serif KR, Wants Sans Variable 사용.
* [ ] 가짜 단어 데이터(Dummy Data) 10개 정도를 화면에 렌더링.

### Phase 2: 핵심 로직 구현 (JavaScript)
* [ ] `app.js`에서 단어 카드를 클릭하면 앞/뒷면이 뒤집히는 애니메이션 구현.
* [ ] 단어의 상태(대기 -> 학습 중 -> 마스터)를 변경하는 버튼 스크립트 작성.
* [ ] 변경된 상태를 `localStorage`에 저장하고 불러오는 기능 완성.

### Phase 3: AI 백엔드 연결 (Cloudflare Workers)
* [ ] Cloudflare 대시보드에서 새로운 Worker 생성.
* [ ] AI API 키(예: Gemini)를 Worker의 환경 변수에 안전하게 저장.
* [ ] `worker.js` 코드 작성: 프론트엔드에서 단어를 보내면 AI가 예문과 뜻을 만들어 JSON으로 응답하는 API 구축.

### Phase 4: 프론트엔드와 AI 연동 및 배포
* [ ] `app.js`에서 `fetch()` 함수를 사용해 Phase 3에서 만든 Worker 주소로 데이터를 요청하고 받아오기.
* [ ] 1000개의 국어 단어 리스트를 `data.js`에 주입.
* [ ] 완성된 폴더를 Cloudflare Pages에 배포하여 실제 인터넷 세상에 오픈!
